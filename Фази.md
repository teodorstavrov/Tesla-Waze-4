# TesRadar — Фази на разработка

Приложение за EV навигация за Tesla браузър (teslaradar.tech).
Технологии: Vite · React 18 · TypeScript · Leaflet · OSRM · Nominatim · Overpass API

---

## ✅ Завършени фази

---

### Фаза 1 — Основа на проекта
**Файлове:** `vite.config.ts`, `src/main.tsx`, `src/app/App.tsx`, `src/lib/leaflet.ts`

- Vite + React 18 + TypeScript scaffold
- Path alias `@/` → `src/`
- Leaflet lazy-import wrapper (`src/lib/leaflet.ts`) — предотвратява SSR crash
- Пълноекранна фиксирана обвивка (`App.tsx`), CSS reset
- Vercel deploy pipeline от `main` branch

---

### Фаза 2 — GPS следене + карта
**Файлове:** `src/components/MapShell.tsx`, `src/features/gps/gpsStore.ts`, `src/features/follow/followStore.ts`, `src/features/gps/useUserPosition.ts`, `src/components/HeadingAvatar.tsx`, `src/components/LocationButton.tsx`

- Leaflet карта, центрирана върху България
- `gpsStore` — module-level pub/sub, не React state (GPS тика на ~1 Hz — нямаме право да ре-рендерираме цялото дърво)
- `followStore` — програматичен mutex за pan/setView без false dragstart
- `HeadingAvatar` — синя точка с компасова стрелка; обновява се без React re-render
- `LocationButton` — таб центрира + активира follow mode
- `ZoomControls` — +/− бутони с 44 px hit area (Tesla touch spec)

### Фаза 3 — Аудио система
**Файлове:** `src/features/audio/audioManager.ts`, `src/features/audio/useAudioUnlock.ts`, `src/components/MuteButton.tsx`

- `audioManager` — Web Audio API beep + speechSynthesis speak, mute toggle
- `useAudioUnlock` — всеки таб върху `<App>` отключва аудио контекста (браузър изисква user gesture)
- `MuteButton` — ляво горе, иконка микрофон/crossed

### Фаза 4 — EV зарядни станции
**Файлове:** `src/features/ev/evStore.ts`, `src/features/ev/EvMarkerLayer.tsx`, `src/features/ev/StationPanel.tsx`, `src/features/ev/types.ts`

### Фаза 23 — GPS step tracking + deviation

### Фаза 24 — Оправяне на React грешка #310
**Файлове:** `src/features/route/routeStore.ts`, `src/features/route/TurnInstruction.tsx`

- **Проблем:** `TurnInstruction` subscribe-ваше и към `gpsStore`, и към `routeStore`. `gpsStore.setPosition()` нотифицира listeners синхронно → React получаваше два update сигнала за един компонент при рендер → `Cannot update a component while rendering a different component`
- **Fix:** `distToNextStepM` се изчислява в `routeStore._onGpsUpdate()` заедно с `remainingM`. `TurnInstruction` subscribe-ва само към `routeStore`. Обновява се само при delta > 20 м.
